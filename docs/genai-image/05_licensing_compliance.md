## 라이선스/컴플라이언스 노트: FAIPL 1.0‑SD (Notices 포함)

> 이 문서는 법률 자문이 아니라, 구현/배포 시 체크리스트 성격의 정리입니다.

### 1) 전제

- 사용할 모델(PerfectDeliberate / Pony Realism / WAI Illustrious SDXL)은 **FAIPL 1.0‑SD** 계열 조건을 가진다고 가정합니다.
- 사용자 설명 기준으로, 해당 라이선스는:
  - **Distribution and Redistribution**에서 Model(또는 Derivatives) 복제/배포를 허용
  - 단, **Notices 조항**에 따라 “사본 수령자에게 라이선스 텍스트 + 대응 소스코드(corresponding source code)” 제공 요구

### 2) 우리가 하려는 행위가 ‘배포’에 해당하는가?

다음 중 하나라도 하면 사실상 배포/재배포로 보는 편이 안전합니다.

- 변환/양자화한 모델을 **Hugging Face에 업로드(공개 또는 비공개 공유)** 하는 경우
- 웹앱에서 사용자가 모델을 다운로드 받도록 **직접 링크/호스팅**하는 경우
- 원본 모델 파일 또는 파생물을 제3자에게 전달하는 경우

### 3) 최소 준수 패키지(권장)

#### 3.1 Hugging Face 모델 저장소(권장 구조)

HF에 변환본(ONNX 번들)을 올릴 때, 번들에 아래를 포함합니다.

- `LICENSE.txt`: FAIPL 1.0‑SD 라이선스 전문(또는 요구되는 라이선스 사본)
- `NOTICE.txt`: 원본 모델/저자/업스트림 링크, 변경 사항(변환/양자화) 명시
- `SOURCE/` 또는 `tools/`:
  - **corresponding source code**로 해석 가능한 “변환/패키징 스크립트” 제공
  - 예: `export_onnx.py`, `quantize.py`, `bundle.py`, `requirements.txt`, 실행 방법
- `README.md`(Model card):
  - 원본 출처 링크(Civitai 페이지)
  - 사용 방법(웹앱에서 요구하는 번들 포맷)
  - 라이선스/제한/면책 고지

#### 3.2 웹앱(UI) 노출(권장)

- “모델 선택/상세” 화면에:
  - 라이선스 이름(FAIPL 1.0‑SD)
  - 원본 출처 링크
  - `LICENSE/NOTICE` 링크(가능하면 HF repo 파일 링크)
- 사용자가 모델을 “다운로드/Import”할 때:
  - 라이선스 동의/고지(최소 링크 제공)

### 4) ‘대응 소스코드’ 범위(실무적 해석 가이드)

라이선스 해석은 케이스가 많아서 확정하기 어렵습니다. 다만 “개발자가 모델을 변환해 배포”하는 시나리오에서
일반적으로 안전한 방향은 아래입니다.

- 최소 제공:
  - 모델을 **어떤 방식으로 변환/최적화/양자화** 했는지 재현 가능한 스크립트
  - 사용한 툴/버전/의존성 명시
- 제공하지 않기 위험한 것:
  - “변환본만 던져두고 생성 방법이 전혀 없음”
